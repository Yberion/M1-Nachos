Detailed Description

Data Structures to emulate an ACIA device.

An ACIA (Asynchronous Communication Interface Adapter) is an asynchronous device (requests return immediately, and an interrupt can be raised later on). It offers two different modes of operation:

1 - A Busy Waiting mode. In this mode, the ACIA does not raise any interrupt when a char transfer is complete. It's up to the driver to read the ACIA state register to detect transfer completion to be able to send the folloging characters.

2- An Interrupt mode. In this mode, the ACIA raises an interrupt when a character has been successfully emitted/received. There are different interrupt levels for emission and reception.
